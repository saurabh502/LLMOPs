Linux Operating System

A comprehensive guide to Linux fundamentals, commands, distributions, and package management.

Linux Fundamentals

What is Linux?

Linux is an open-source operating system kernel developed by Linus Torvalds in 1991. It's the core of many operating systems, often called "Linux distributions."

  • Developed by Linus Torvalds in 1991
  • Core component of many operating systems (distributions)
  • Powers everything from smartphones to supercomputers
Linux is technically just the kernel, but commonly refers to the entire OS (kernel + utilities + software)

Open Source

One of Linux's defining characteristics is its open-source nature. The source code is freely available for anyone to view, modify, and distribute.

Key Aspects:

  • Source code is freely available
  • Users can modify and customize
  • Freedom to distribute changes
  • Promotes collaboration and innovation

Did You Know?

The open-source model has led to Linux being used in over 90% of the world's supercomputers and forming the basis for Android, the world's most popular mobile OS.

Key Features

Linux offers a robust set of features that make it powerful and versatile for various computing needs.

Portability

Runs on various hardware from embedded devices to mainframes.

Multitasking

Can run multiple programs simultaneously with efficient resource management.

Multi-user

Multiple users can access the system simultaneously with proper permissions.

Security

Robust permission system, SELinux, and regular security updates.

Customizable

Highly configurable to meet specific user needs and preferences.

CLI & GUI

Powerful command-line interface and optional graphical interfaces like GNOME, KDE.

Linux vs. Unix

While Linux was inspired by Unix, there are important differences between these operating systems.

Aspect Linux Unix
Origin Developed by Linus Torvalds in 1991 Developed at AT&T Bell Labs in 1969
License GNU GPL (Open Source) Mostly proprietary
Cost Free Commercial (expensive)
Development Community-driven Vendor-specific
Variants Many distributions (Ubuntu, Fedora, etc.) AIX, HP-UX, Solaris, etc.

Compatibility Note

Despite differences, Linux maintains high compatibility with Unix standards (POSIX), allowing many Unix programs to run on Linux with little or no modification.

Core Components

Kernel

The kernel is the core of the Linux operating system, acting as the bridge between software and hardware.

  • Manages system resources (CPU, memory, devices)
  • Provides an interface for user-level programs
  • Handles security and access control
  • Manages device drivers and hardware communication

Kernel Types

Linux kernels come in different flavors: monolithic (traditional), microkernel (experimental), and hybrid. Most distributions use the monolithic kernel which includes all core functionality.

Shell

The shell is a command-line interpreter that provides a user interface to interact with the kernel and execute commands.

Common Shells:

  • Bash (Bourne Again SHell) - Default on most Linux distributions
  • Zsh (Z Shell) - Powerful alternative with advanced features
  • Fish (Friendly Interactive SHell) - User-friendly with syntax highlighting
  • Ksh (Korn SHell) - Compatible with Bash but with some differences

Example Bash Command:

user@linux:~$ echo "Hello, World!"
Hello, World!

File System

Linux organizes data into files and directories using a hierarchical structure (tree-like).

Key Characteristics:

  • Everything is a file - Including devices, directories, and sockets
  • Single root - All files and directories stem from / (root)
  • Permissions - Fine-grained control over file access
  • Journaling - Prevents data corruption (ext4, XFS, Btrfs)

Common Linux File Systems:

ext4
XFS
Btrfs

Directory Structure

Linux follows the Filesystem Hierarchy Standard (FHS) with directories like /bin (essential binaries), /etc (configuration files), /home (user directories), /var (variable data), and /usr (user programs).

Graphical User Interface (GUI)

While Linux is famous for its command-line interface, it also offers various graphical environments for user-friendly interaction.

Desktop Environments:

  • GNOME - Modern, clean interface (default on Ubuntu, Fedora)
  • KDE Plasma - Feature-rich, highly customizable
  • XFCE - Lightweight, ideal for older hardware
  • LXQt - Ultra-lightweight for minimal systems

Window Managers:

i3 - Tiling window manager for keyboard-centric workflow
Awesome - Highly configurable, extensible with Lua
Openbox - Lightweight stacking window manager
Compiz - Known for visual effects and desktop cube

Display Servers

Linux GUIs rely on display servers like X11 (traditional) or Wayland (modern). Wayland offers better security and performance but may lack some compatibility with older applications.

System Utilities

Linux includes a comprehensive set of tools for system administration and maintenance.

Process Management

  • ps - List processes
  • top - Dynamic process viewer
  • kill - Terminate processes
  • nice - Adjust process priority

Networking

  • ifconfig - Network interface config
  • ping - Test network connectivity
  • netstat - Network statistics
  • ssh - Secure remote access

Disk Management

  • df - Disk space usage
  • du - Directory space usage
  • mount - Mount filesystems
  • fsck - Filesystem check

System Monitoring

  • uptime - System uptime
  • free - Memory usage
  • vmstat - Virtual memory stats
  • dmesg - Kernel messages

Systemd

Modern Linux distributions use systemd as their init system and service manager, providing commands like systemctl (control services), journalctl (view logs), and timedatectl (manage time).

Linux Distributions

Linux Distributions (Distros)

A Linux distribution is a complete operating system based on the Linux kernel, packaged with software for various purposes.

Popular Linux Distributions:

Debian

Stable, used as a base for many other distros. Known for its strict free software policy.

Stable Universal APT
Ubuntu

User-friendly, popular for desktops and servers. Based on Debian with regular releases.

User-friendly LTS Snap
Fedora

Cutting-edge, community-driven. Often includes newest technologies. Sponsored by Red Hat.

Innovative DNF RPM
RHEL

Red Hat Enterprise Linux. Commercial, stable, with long-term support for enterprise use.

Enterprise Stable YUM
Arch Linux

Highly customizable, rolling release. For advanced users who want control over their system.

Rolling Minimal Pacman
openSUSE

Versatile distro with both stable (Leap) and rolling (Tumbleweed) versions. Known for YaST tool.

YaST RPM ZYpp

Choosing a Distribution

The right distribution depends on your needs: Ubuntu for beginners, Fedora for developers, Debian for stability, Arch for customization, RHEL/CentOS for enterprise servers. Consider factors like hardware support, software availability, and community support.

Bash Commands

Basic Commands

These are fundamental commands every Linux user should know for navigating and managing the filesystem.

Command Description Example
pwd Print working directory pwd
ls List files and directories ls -l
cd Change directory cd /home/user
mkdir Create directory mkdir new_folder
rmdir Remove empty directory rmdir empty_folder
touch Create empty file touch new_file.txt
rm Remove file rm file.txt
cp Copy file or directory cp file.txt backup/
mv Move/rename file or directory mv old.txt new.txt
cat Display file content cat file.txt
less View file content page by page less long_file.txt
head Display beginning of file head -n 10 file.txt
tail Display end of file tail -f logfile.log
echo Display text echo "Hello"
man Display manual page man ls
history Show command history history | grep "apt"

Command Tips

  • Use Tab for auto-completion of commands and filenames
  • Press Ctrl+C to stop a running command
  • Use Ctrl+R to search through command history
  • Append --help to most commands for quick help (e.g., ls --help)

File Permissions

Linux uses a robust permission system to control access to files and directories.

Understanding Permissions:

Permission Types:
  • r Read (4)
  • w Write (2)
  • x Execute (1)
Permission Groups:
  • u User/Owner
  • g Group
  • o Others

Key Commands:

chmod
Change file permissions
chmod 755 script.sh
chown
Change file owner
chown user:group file
chgrp
Change file group
chgrp team file

Permission Examples

Numeric Symbolic Meaning
777 rwxrwxrwx All permissions to everyone
755 rwxr-xr-x Owner: rwx, Group/Others: r-x
644 rw-r--r-- Owner: rw-, Group/Others: r--
600 rw------- Owner: rw-, no permissions for others

Redirection and Piping

Linux provides powerful tools for redirecting input/output and chaining commands together.

Redirection Operators:

>
Redirect output to a file (overwrite)
ls -l > filelist.txt
>>
Redirect output to a file (append)
echo "New line" >> file.txt
<
Redirect input from a file
sort < unsorted.txt
2>
Redirect error output
command 2> error.log

Piping:

|
Pipe output of one command to input of another
ls -l | grep "myfile"
Common pipeline commands:
  • grep Search for patterns
  • sort Sort lines
  • uniq Filter duplicate lines
  • wc Count words/lines
  • awk Pattern scanning and processing
  • sed Stream editor for filtering/transforming

Advanced Piping

You can chain multiple commands together with pipes. For example: cat logfile.txt | grep "ERROR" | sort | uniq -c | sort -nr This command sequence: reads a log file, filters for ERROR lines, sorts them, counts unique occurrences, then sorts by count in reverse order.
